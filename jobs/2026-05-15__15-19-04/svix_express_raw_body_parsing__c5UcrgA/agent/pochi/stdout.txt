The Svix webhook verification issue has been fixed in `/home/user/app/server.js`. Here's a summary of what was wrong and what changed:

## Root Cause

`app.use(express.json())` was registered **globally** before the `/webhook` route. This means Express fully deserializes the request body into a plain JS object before the route handler runs. Svix's `wh.verify()` requires the **original raw bytes** to recompute the HMAC — passing a JS object instead causes verification to always fail.

## Fix Applied

```js
// BEFORE — global JSON parser breaks Svix verification
app.use(express.json());

app.post('/webhook', (req, res) => {
    const msg = wh.verify(req.body, req.headers); // ❌ req.body is a plain object
});
```

```js
// AFTER — raw body parser scoped to /webhook only
app.post('/webhook', express.raw({ type: 'application/json' }), (req, res) => {
    const msg = wh.verify(req.body, req.headers); // ✅ req.body is a Buffer
});

// JSON parsing still works for every other route
app.use(express.json());
app.post('/api/data', (req, res) => { res.json(req.body); });
```

**Key points:**
- `express.raw({ type: 'application/json' })` is applied as **route-level middleware** only on `/webhook`, so it captures the raw Buffer before any JSON parsing occurs.
- `express.json()` is registered **after** the `/webhook` route definition, so it applies to all other routes (`/api/data`, etc.) without interfering with webhook verification.
- Svix's `Webhook.verify()` accepts either a `Buffer` or a `string`, so passing the raw Buffer directly is correct.
